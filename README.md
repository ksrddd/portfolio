# 🛡️ Cybersecurity Portfolio Project Guide

ยินดีต้อนรับสู่คู่มือการพัฒนา **Cybersecurity Portfolio** ครับ! 
เอกสารนี้ถูกสร้างขึ้นมาเพื่อให้คุณใช้อ่านทบทวนทำความเข้าใจว่า "โครงสร้างของเว็บที่เราสร้างนั้นทำงานยังไงบ้าง" โดยเราจะใช้เอกสารนี้เป็น "สมุดจด" (Living Document) ที่จะอัพเดทเรื่อยๆ ตามฟีเจอร์ที่เราเพิ่มเข้าไปในอนาคตครับ

---

## 🛠️ เทคโนโลยีที่เราใช้ (Tech Stack)
1. **Frontend**: `Vue 3` + `Nuxt.js 3` + `Tailwind CSS` (ทำหน้าเว็บ เน้นความสวยงาม ธีมแฮกเกอร์ และ SEO)
2. **Backend**: `Node.js` + `Express` (ทำเซิร์ฟเวอร์ส่งข้อมูล)
3. **Database (อนาคต)**: `MySQL` (เก็บข้อมูลจริง)

---

## 📂 ฝั่งโฮสต์หลัก (Backend)
โฟลเดอร์: `/backend`
ทำหน้าที่เป็น "ผู้จัดการข้อมูล" มันไม่มีหน้าตา (UI) แต่จะรอรับคำสั่งผ่าน URL (API) แล้วคายข้อมูลออกไปให้ Frontend เอาไปโชว์

### ไฟล์อัพเดทล่าสุด
- **`index.js`** 
  - จุดเริ่มต้นของแอป Express ของเรา เปิดเซิร์ฟเวอร์ที่พอร์ต `5000`
  - มีการใช้ `cors()` เพื่อยอมให้ Frontend ของเรายิงข้อมูลข้ามพอร์ตมาขอข้อมูลได้
- **`src/config/db.js`**
  - ไฟล์เตรียมการสำหรับเชื่อมต่อ **MySQL Database** ไว้ดึงตารางโปรเจคของจริง (ตอนนี้ตั้งค่ารอไว้ก่อน)
- **`src/routes/projects.js`**
  - สมุดบัญชีจด URL ว่าถ้ามีคนเข้า API เส้นไหน จะส่งไปให้ฟังก์ชันไหนจัดการ
  - มี 2 เส้นทาง: 
    1. `/` -> ยิงไปหา `getProjects` (ดึงทุกอย่าง)
    2. `/:id` -> ยิงไปหา `getProjectById` (ขอดึงแค่ 1 อันตามตัวเลข id)
- **`src/controllers/projectController.js`**
  - **สมองหลัก** ของ Backend ที่ทำหน้าที่ประมวลผล เช่น ไปหาข้อมูลจำลอง (Mock Data) จาก Array ของเรา
  - `getProjects()` เอาไว้ส่งรายชื่อทั้งหมดไปโชว์ในหน้าแรกของเว็บ
  - `getProjectById()` เอาไว้จับคู่ `id` ว่า User จะดูโปรเจคไหน แล้วส่งเนื้อหาแบบเต็มๆ กลับไปให้

## 💻 ฝั่งหน้าจอผู้ใช้ (Frontend)
โฟลเดอร์: `/frontend/cyber-portfolio`
ทำหน้าที่วาดหน้าจอ สร้างโครงสร้าง CSS สวยๆ และดึง API จาก Backend มาแสดงผลให้อัตโนมัติ

### โครงสร้างการทำงานของ Nuxt 3
- **`nuxt.config.ts`**
  - ศูนย์ควบคุมของ Nuxt ที่เราเอาไว้เปิดใช้ TailwindCSS และใส่ฟอนต์ Google Fonts (Fira Code แนว Hacker)
- **`app.vue`**
  - โครงสร้าง "หน้ากากหลัก" ตอนนี้เราใส่แท็ก `<NuxtPage />` ลงไป เพื่อบอกให้มันไม่ต้องสนใจวาดหน้าจอที่นี่ แต่ให้ไปอ่านไฟล์จากโฟลเดอร์ `pages` แทน!
- **`assets/css/main.css`** & **`tailwind.config.js`**
  - แหล่งเก็บพลัง CSS โทนสี Cyber Green (`#00ff41`) และเอฟเฟกต์ตัวอักษรสั่นๆ (Glitch) ตารางสีเขียวด้านหลัง
- **`pages/index.vue`**
  - **หน้าแรกของเว็บ (`http://localhost:3000/`)**
  - วาดหน้าจอแนะนำตัวเบื้องต้น และใช้คำสั่ง `useFetch` วิ่งไปหา Backend เพื่อขอข้อมูลรายชื่อโปรเจคมาแสดงผล
  - ใส่ฟังก์ชัน `<NuxtLink :to="...">` ครอบรายชื่อไว้ เพื่อให้กดคลิกไปเจาะลึกแต่ละงานได้
- **`pages/projects/[id].vue`**
  - **หน้ารายละเอียด (`http://localhost:3000/projects/1`)**
  - สิ่งที่อยู่ในก้ามปู `[id]` คือระบบ Dynamic Routing สุดล้ำของ Nuxt
  - มันจะไปแอบดู URL ว่า User กดเลขอะไรมา (เช่น `/projects/1`) ตัวแปร `id` จะกลายเป็น `1`
  - จากนั้นมันจะเอาเลข `1` นี้ ไปยิงถาม Backend รายบุคคล (`/api/projects/1`) 
  - สุดท้ายนำข้อมูลที่ได้ เช่น `full_detail` ไปแสดงผลสวยๆ โชว์ให้ผู้ใช้เห็น!

## 🚀 วิธีการรันโปรเจคเพื่อทดสอบ

หากคุณจะเริ่มงานหรือลองรันตอนเช้า ต้องเปิด 2 Terminal ควบคู่กันไปครับ:

**Terminal 1 (เริ่ม Backend)**
```bash
cd backend
npm run dev
```

**Terminal 2 (เริ่ม Frontend)**
```bash
cd frontend/cyber-portfolio
npm run dev
```

เปิดเบราว์เซอร์ไปที่: `http://localhost:3000`
